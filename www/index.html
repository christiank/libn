→ → libn

libn is a small library to help make writing C a little more fun. It is a portable, lightweight and BSD-licensed alternative to GLib and GObject, or something like a less cumbersome version of Boost, but for straight C instead of C++.

Under the hood, libn uses mruby to make object-oriented programming in C actually doable. You could think of libn as a collection of convenience routines on top of mruby, but mruby is really just an implementation detail.

Hello, libn

With libn, you can write code like this:


/*
 * hello.c
 */

#include <stdio.h>
#include <stdlib.h>
#include <libn.h>

int
main(void)
{
  nstringhash *person;

  libn_start();
  person = nstringhash_new();
  nstringhash_set(person, "firstname", "Mildred");
  nstringhash_set(person, "lastname", "Haversham");
  printf("%s\n", nstringhash_inspect(person));

  nstringhash_free(person);
  libn_finish();
  return EXIT_SUCCESS;
}

And then compile it like this:


$ clang -Wall \
  -o hello \
  hello.c \
  -lmruby -lm -ln

What's wrong with Glib and GObject?

  • Unnecessary portability typedefs.
  • THEY'RE WAAAY TOO HUGE.
  • Extremely portable, yet nearly impossible to build on some systems (OS X, most notably).
  • Documentation is pretty good, but you have to use devhelp(1) in order to read it :(
  • GObject really does NOT make any sense.
  • You can't statically link against them.
  • You can't simply copy the source code into your project. (Well, you can try, but you'll always fail.)
